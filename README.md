# Paint

## Introduction

Projet de paint en Java dans le cadre de l'IFT 232 (Conception Orientée Objet).

## GIT

 - La branche [master] n'est utilisée que pour les 'releases'. Les push / merge dans cette branche doivent être accompagnés de tag correspondant à un numéro de version __STABLE__. Les commits y sont donc très peu nombreux.
 - La branche [develop] sert de piscine commune. On y dépose nos fonctionnalités (trivialement _fonctionnelles_). Après avoir déposé une fonctionnalité, il faut faire un pull request pour le master.
 - Chaque couple de _pair-programming_ doit créer sa branche pour y implanter une fonctionnalité. Le nom de la branche doit contenir les noms des membres du couple.
